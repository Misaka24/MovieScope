# IMDb Provider 拆分审查

## 审查结论

本轮未发现阻断提交的问题。Just One API 的 IMDb 调用已收束为 Top 250 和新闻，其他 IMDb 评分由 `imdbapi.dev` 提供。真实接口回归中发现并修复了批量接口的故障放大问题：单个坏 IMDb ID 不再导致同批其他条目全部回退。

## 验证结果

- `npm run typecheck` 通过。
- `npm run build` 通过。
- `GET /api/v1/health` 返回 PGlite 数据库健康。
- `GET /api/v1/home` 的九个上游来源均成功，返回 5 个 Hero、四个 6 条影视分区和 3 条新闻。
- 正在上映、IMDb Top 250 和热门电影的 IMDb 评分实测均为 `6/6`；热门剧集在隔离不可用 ID 后由 `1/6` 提升为 `4/6`，其余条目明确使用 TMDB 回退。
- SQL 实测 TTL：Top 250 为 `604800` 秒，新闻和 `imdbapi.dev` 批量详情均为 `86400` 秒。
- 已删除旧 Just One 单片评分活动缓存；历史同步日志保留。新缓存键只包含 IMDb ID，不包含第三方令牌。

## 剩余风险

- `imdbapi.dev` 会对个别尚未同步或上游异常的 IMDb ID 返回 500；递归拆分可保护同批有效数据，但首次遇到坏 ID 时会增加少量请求。
- Hero 每次从 TMDB 热门内容随机抽取，IMDb 覆盖率会随抽样结果变化；没有 IMDb 数据时继续显示真实 TMDB 分数，不伪造来源。
