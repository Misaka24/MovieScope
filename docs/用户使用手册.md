# MovieScope 用户使用手册

> 适用范围：MovieScope 用户主站、个人中心与公开主页
>
> 当前实现：Vue 3 + Vite、Node.js API、MySQL

## 1. 身份与功能

| 身份 | 可用功能 |
| --- | --- |
| 未登录访客 | 浏览首页、搜索、探索、榜单、影视与影人详情、新闻、公开用户主页和公开短评 |
| 注册用户 | 包含访客功能，并可保存观影状态、收藏、评分、短评、个人资料、浏览历史和搜索历史 |

管理员也可以使用全部用户功能，其后台操作参见《管理员操作手册》。

## 2. 页面入口

本地开发环境默认地址为 `http://127.0.0.1:5173`。

| 页面 | 地址 | 用途 |
| --- | --- | --- |
| 首页 | `/` | 浏览热门、上映、高分、剧集和影坛动态 |
| 搜索 | `/search` | 搜索电影、剧集和影人 |
| 探索 | `/explore` | 按类型、年份、评分、语言、平台等条件筛选 |
| 影视详情 | `/title/movie/:id`、`/title/tv/:id` | 查看资料、评分、演职员、图片、观看平台并记录互动 |
| 影人详情 | `/person/:id` | 查看人物资料和作品 |
| 新闻 | `/news` | 浏览和搜索影视资讯 |
| 登录 | `/auth?mode=login` | 使用用户名或邮箱登录 |
| 注册 | `/auth?mode=register` | 创建账号 |
| 个人中心 | `/me` | 管理个人资料和全部用户数据 |
| 公开主页 | `/users/:username` | 查看用户允许公开的资料与记录 |

路由中的 `:id` 和 `:username` 会在点击内容时自动生成，通常不需要手动填写。

## 3. 注册、登录与退出

### 3.1 注册

1. 点击顶栏“注册”，或访问 `/auth?mode=register`。
2. 填写用户名、显示名称、邮箱和密码。
3. 注册成功后系统自动登录，并进入个人中心。

字段规则：

| 字段 | 规则 |
| --- | --- |
| 用户名 | 3～32 个字符，必须唯一；会成为公开主页地址的一部分 |
| 显示名称 | 必填，最长 60 个字符，可在个人中心修改 |
| 邮箱 | 必须符合邮箱格式且未被注册 |
| 密码 | 至少 8 个字符 |

当前版本尚未提供邮箱验证、找回密码和第三方账号登录。

### 3.2 登录

1. 点击顶栏“登录”，或访问 `/auth?mode=login`。
2. 输入用户名或邮箱。
3. 输入密码并提交。

登录状态使用 HttpOnly Cookie 保存，密码不会存入前端 localStorage。默认会话有效期为 30 天，可由服务器环境变量调整。

同一账号标识和 IP 地址在 15 分钟内连续登录失败 8 次后，后续尝试会暂时返回 `429`。此时应停止重试，等待限流窗口结束。

### 3.3 退出与修改密码

点击顶栏头像，在账号菜单中选择“退出登录”，服务端会撤销当前会话。

在个人中心成功修改密码后，该账号现有会话会全部撤销，用户需要使用新密码重新登录。在公共设备使用后应主动退出。

## 4. 搜索、探索与历史

### 4.1 搜索

顶栏和搜索页支持检索电影、剧集和影人。登录用户发起有效关键词搜索时，系统会把关键词、搜索类型和时间写入 MySQL。

个人中心展示最近 100 条搜索历史。点击历史关键词可以重新发起搜索，清空搜索历史不会影响观影记录。

### 4.2 浏览历史

登录状态下打开影视或影人详情时会写入浏览历史。个人中心展示最近 100 条，后端为每位用户保留最近 200 条。清空浏览历史不会删除想看、评分、收藏或短评。

### 4.3 探索筛选

探索页根据真实接口能力提供媒体类型、年份、最低评分、语言、排序方式和观看平台等筛选。选择观看平台时，平台标识和观看地区会真正传入查询请求。

筛选结果受上游数据收录和地区版权信息影响。无结果时可放宽年份、评分或平台条件。

## 5. 影视详情与用户互动

详情页尽可能展示中文标题、原始标题、简介、类型、年份、时长、演职员、剧照、评分人数、官方网站和观看平台。上游未提供的字段会显示“暂无”或不展示。

### 5.1 观影状态

登录后可将电影或剧集标记为“想看”“在看”或“看过”。同一影视只有一个当前状态，切换状态会更新原记录，不会生成重复数据。

### 5.2 收藏

收藏和取消收藏只改变收藏状态，不会覆盖已有的观影状态、评分和短评。

### 5.3 评分

用户评分范围为 0.5～10 分，数据库保留 1 位小数。页面中的 IMDb 或 TMDB 媒体评分属于外部数据，“我的评分”属于 MovieScope 用户数据，二者互不覆盖。

### 5.4 短评

在“你的互动”区域可提交最长 5000 个字符的短评，并标记是否包含剧透。公开短评会显示在相应影视详情页。

以下短评不会向其他访客展示：

- 用户主页设为“仅自己”；
- 用户账号被停用；
- 短评被管理员隐藏；
- 用户删除对应记录或清空短评。

剧透标记只提供提示，不会自动识别或改写内容。

### 5.5 分享和观看平台

分享按钮默认复制该作品的 IMDb 链接。没有 IMDb 链接时，功能可能受数据完整度影响。

当接口提供指定地区的真实观看平台数据时，页面会显示平台入口。平台可用性由版权地区决定，MovieScope 不直接提供视频播放服务。

## 6. 个人中心

登录后访问 `/me`，或从顶栏用户菜单进入。个人中心包含：

| 分区 | 内容 |
| --- | --- |
| 动态记录 | 最近更新的观影、评分、短评和收藏记录 |
| 想看、在看、看过 | 按当前观影状态分类 |
| 评分与短评 | 已评分或已填写短评的影视 |
| 收藏 | 已收藏的影视 |
| 浏览历史 | 最近访问的电影、剧集和影人 |
| 搜索历史 | 最近搜索的关键词与类型 |
| 编辑资料 | 显示名称、简介、头像、背景、主页可见性和密码 |

顶部统计来自 MySQL 中的真实记录，包括已评分、短评、想看和收藏数量。

点击影视卡片会进入详情页。卡片上的“删除记录”会删除该影视的全部个人数据，包括观影状态、收藏、评分和短评，当前页面不提供撤销。

## 7. 个人资料与隐私

用户可以修改：

- 显示名称；
- 个人简介，最长 500 个字符；
- 头像 URL；
- 背景图 URL；
- 主页可见性；
- 登录密码。

当前头像和背景使用图片 URL，不提供本地文件上传。应填写可公开访问的 HTTPS 图片地址。

### 7.1 公开主页

公开主页地址为 `/users/<用户名>`。主页公开时，其他人可查看公开资料、统计、观影状态、评分和已公开短评。

公开接口不会返回邮箱、密码哈希、会话、搜索历史或管理员信息。

### 7.2 私密主页

设为“仅自己”后：

- 本人登录后仍可访问；
- 其他访客访问会收到无权限提示；
- 用户短评不会进入影视详情页的公开短评列表；
- 原有数据不会删除，重新公开后可以恢复展示。

## 8. 数据保存规则

用户数据全部保存到 MySQL，不依赖浏览器 localStorage：

| 数据 | 数据表 |
| --- | --- |
| 账号和资料 | `users` |
| 登录会话 | `user_sessions` |
| 状态、收藏、评分、短评 | `user_media_entries` |
| 浏览历史 | `user_browse_history` |
| 搜索历史 | `user_search_history` |

外部影视资料可能随 API 更新或缓存刷新而变化，但不会覆盖用户自己的状态、评分和短评。

## 9. 常见问题

| 现象 | 原因与处理 |
| --- | --- |
| 提示“请先登录” | 会话不存在或已失效，重新登录后再操作 |
| 用户名或密码错误 | 核对账号，避免连续快速重试触发限流 |
| 返回 `429` | 登录失败次数过多，等待 15 分钟统计窗口结束 |
| 账号已停用 | 联系管理员核查账号状态 |
| 主页未公开 | 对方设置为私密，需由主页所有者修改 |
| 评分显示“暂无” | 外部数据源未收录或暂时不可用，不影响用户评分保存 |
| 图片无法显示 | 图片 URL 失效、防盗链或网络不可达 |
| 服务器返回的数据格式无效 | API 未启动、代理端口错误或服务返回非 JSON 内容 |
| `502` 或服务器请求失败 | 上游数据源暂时不可用，稍后重试；已保存用户数据不受影响 |

## 10. 用户侧验收步骤

1. 注册账号并确认自动登录。
2. 退出后分别使用用户名和邮箱登录。
3. 搜索影视并进入详情，确认搜索和浏览历史产生记录。
4. 设置想看后改为看过，确认仅有一条记录且状态更新。
5. 收藏、评分、提交短评，刷新后确认数据仍存在。
6. 取消收藏，确认状态、评分和短评未被清除。
7. 查看公开主页，确认不显示邮箱等私密信息。
8. 将主页设为私密，退出登录后确认其他访客无法访问。
9. 修改密码，确认会话失效、旧密码不能登录。
10. 清空两类历史，确认观影记录仍保留。
