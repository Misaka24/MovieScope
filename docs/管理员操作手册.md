# MovieScope 管理员操作手册

> 适用范围：MovieScope 管理后台、管理员账号维护、短评审核和运行状态核查
>
> 后台地址：`/admin`

## 1. 权限与职责

MovieScope 当前有两种角色：

| 角色 | 权限 |
| --- | --- |
| `user` | 使用主站、个人中心和公开主页 |
| `admin` | 包含用户权限，并可访问后台概览、用户管理、短评审核和日志 |

管理员后台现在包含总览、统计分析、用户管理、分组与权限、内容审核、API 管理、缓存管理、日志中心和站点设置九个工作区。管理员可以：

- 查看用户、观影记录、评分、短评、会话和缓存汇总；
- 按用户名、邮箱或显示名称搜索用户；
- 修改用户角色；
- 停用和恢复用户；
- 隐藏或恢复短评；
- 查看管理员审计、数据源同步和登录安全日志；
- 维护 API Provider 的地址、启停、超时、额度说明和健康检查；
- 查询、添加、覆盖和清理 MySQL API 缓存；
- 管理注册、维护模式、站点公告和短评自动公开策略。

### 1.1 权限模型

账号字段 `users.role='admin'` 只表示允许进入后台，具体功能由 RBAC 决定：

- 管理员分组提供权限继承；
- 一个管理员可以加入多个分组；
- 用户级覆盖可以对单项权限明确“允许”或“拒绝”；
- 拒绝覆盖优先移除分组继承的权限；
- 后台菜单按有效权限动态显示；
- 每个后台 API 仍在服务端执行权限校验。

内置分组包括超级管理员、内容审核员、接口运维员和数据分析员。超级管理员始终拥有当前全部 15 项后台权限，保存时不会接受权限缩减；系统分组不能删除。

管理员当前不能在后台读取密码、查看会话 Token 明文、编辑短评正文、直接重置密码或物理删除账号。密码以随机盐 `scrypt` 哈希保存，无法还原为明文。

## 2. 运行前提

- Node.js 和项目依赖已安装；
- MySQL 8 可连接，数据库已创建；
- 项目根目录有未提交到 Git 的 `.env`；
- 数据库账号具有迁移和业务读写权限；
- 前端和 API 服务均已启动。

推荐从项目根目录执行：

```powershell
npm run dev:all
```

默认地址：

- 主站：`http://127.0.0.1:5173`
- API：`http://127.0.0.1:8787`
- 后台：`http://127.0.0.1:5173/admin`

分别启动时使用：

```powershell
npm run api
npm run dev:web
```

若端口 `8787` 报 `EADDRINUSE`，说明已有 API 进程监听该端口。应确认并复用现有服务，或正常结束旧进程后再启动。

## 3. 环境变量

| 变量 | 用途 | 默认/说明 |
| --- | --- | --- |
| `MYSQL_URL` | MySQL 完整连接串 | 配置后优先于分项参数 |
| `MYSQL_HOST` | MySQL 地址 | `127.0.0.1` |
| `MYSQL_PORT` | MySQL 端口 | `3306` |
| `MYSQL_USER` | 数据库用户 | 本地默认 `root`，生产环境应使用专用低权限账号 |
| `MYSQL_PASSWORD` | 数据库密码 | 只放在安全环境配置，不得写入代码或文档 |
| `MYSQL_DATABASE` | 数据库名 | `moviescope` |
| `MYSQL_CONNECTION_LIMIT` | 连接池上限 | `10` |
| `API_PORT` | API 端口 | `8787` |
| `WEB_ORIGIN` | 允许写请求的前端来源 | `http://127.0.0.1:5173` |
| `SESSION_DAYS` | 登录会话天数 | `30` |
| `TMDB_ACCESS_TOKEN` | TMDB 服务端令牌 | 不得暴露给浏览器 |
| `JUSTONE_API_TOKEN` | Top 250、新闻保留接口令牌 | 不得暴露给浏览器 |
| `IMDB_API_DEV_BASE_URL` | IMDb API 地址 | 默认 `https://api.imdbapi.dev` |
| `API_TIMEOUT_MS` | 第三方请求超时 | 示例配置为 30000 毫秒 |

生产环境的 `WEB_ORIGIN` 必须与实际前端协议、域名和端口完全一致，否则登录、资料保存和审核等写请求会被 Origin 校验拒绝。

## 4. 数据库与迁移

迁移文件位于 `db/migrations/`，后端按文件名顺序执行全部 `*.mysql.sql` 文件。

当前主要数据表：

| 表 | 用途 |
| --- | --- |
| `users` | 用户资料、角色、状态和主页可见性 |
| `user_sessions` | 登录会话及撤销、过期状态 |
| `auth_login_attempts` | 登录成功/失败和限流依据 |
| `user_media_entries` | 观影状态、收藏、评分、短评和审核状态 |
| `user_browse_history` | 浏览历史 |
| `user_search_history` | 搜索历史 |
| `admin_audit_logs` | 管理员操作审计 |
| `api_cache` | 第三方 API SQL 缓存 |
| `provider_sync_logs` | 数据源同步和错误日志 |

迁移不会主动清空业务数据，但生产环境执行结构升级前仍应完成并验证 MySQL 备份。

## 5. 创建管理员

系统没有默认管理员，也不会把第一个注册用户自动设为管理员。

1. 在主站注册专用管理账号。
2. 在项目根目录执行：

```powershell
npm run admin:promote -- <用户名或邮箱>
```

示例格式：

```powershell
npm run admin:promote -- admin@example.com
```

3. 提升成功后重新登录。
4. 确认顶栏用户菜单出现“管理后台”，并访问 `/admin`。

提升脚本会把目标账号设为 `admin`、恢复为 `active`，并加入“超级管理员”组。用户名或邮箱匹配不区分英文大小写。

管理账号建议：

- 与公开日常账号分开；
- 使用独立长密码；
- 不共享 Cookie 和浏览器配置；
- 至少保留两个受控管理员；
- 生产环境 Web 服务不要长期使用 MySQL `root`。

## 6. 进入后台

管理员登录后从顶栏头像菜单进入后台，或直接访问 `/admin`。前端会检查当前用户角色，后端对每个 `/api/v1/admin/*` 请求再次鉴权。

普通用户访问后台会被跳回主站；直接请求后台 API 返回 `403`。账号被停用后，已有会话也不会继续获得权限。

## 7. 数据概览与统计分析

“数据概览”显示 MySQL 实时汇总：

| 指标 | 含义 |
| --- | --- |
| 注册用户 | 用户总数和管理员数量 |
| 停用用户 | 状态为 `suspended` 的账号数 |
| 观影记录 | 用户影视互动记录总数 |
| 评分数量 | 包含用户评分的记录数 |
| 站内短评 | 包含短评正文的记录数 |
| 已隐藏短评 | 审核状态为 `hidden` 的数量 |
| 有效会话 | 未撤销且未过期的会话数 |
| 24 小时接口错误 | 最近 24 小时 Provider 失败数 |
| SQL 缓存 | 各 Provider 缓存总数和有效数 |

概览用于日常观察，不替代正式监控。接口错误持续增加时，应进入“数据源日志”检查 Provider、操作、耗时和错误信息。

统计分析工作区进一步提供：

- 近 30 天每日注册趋势；
- 用户角色与状态结构；
- 电影、剧集和观影状态分布；
- 搜索类型与搜索时段；
- 各 Provider 七日调用量、成功率、平均与最大延迟；
- 站内热门作品、热门搜索和 14 天活动柱状图。

统计分析页还提供“影视数据分析工作台”，所有指标都按具体电影或剧集聚合，而不是只有全站总数。管理员可以按电影/剧集、原始语言、类型标签、用户评分范围筛选，并切换记录总数、参与用户、收藏、在看、看过、评分数量、用户均分和短评数量生成排行榜。结果表可直接查看具体影视的想看、在看、看过、收藏、评分人数、用户均分和短评数，并可点击片名进入详情页。

统计维度的口径如下：

- “搜索类型”来自 `user_search_history.search_type`。用户搜索时选择“全部、电影、剧集、影人”等范围，后端会连同关键词和时间写入搜索历史；它统计的是搜索行为的范围，不是影视自身类型。
- “影视状态分布”来自 `user_media_entries.watch_status`，表示用户对某部具体影视设置的“想看、在看、看过”。“无观影状态”表示该条记录只有收藏、评分或短评，没有选择观影状态。
- 类型标签、原始语言和来源国家会在用户首次对影视执行在看、收藏、评分等操作时，随详情页数据写入 `user_media_entries`，供后续多维统计使用。历史记录如果尚未保存这些字段，会显示为“未记录”，在用户再次操作该影视后自动补齐。

## 8. 用户管理

### 8.1 搜索

搜索框支持用户名、邮箱和显示名称。输入关键词后点击“搜索”或按 Enter；留空会返回默认用户列表。

### 8.2 修改角色

角色可在“用户”和“管理员”之间切换。修改立即写入 MySQL，并记录审计日志。

系统禁止当前管理员移除自己的管理员角色。若需降级某位管理员，应由另一名管理员执行。

### 8.3 停用和恢复

状态可设为“正常”或“停用”。停用后：

- 用户不能建立有效登录会话；
- 已有会话在后续鉴权时失效；
- 用户不会进入公开短评查询；
- 原有数据不会物理删除，恢复后可继续使用。

系统禁止管理员停用自己的当前账号。操作前应核对用户名、邮箱和显示名称，防止误操作同名用户。

### 8.4 用户详情和批量操作

用户详情抽屉提供账号资料、风险等级、管理员备注、观影统计、最近互动和最近 30 条会话。具有 `users.manage` 权限的管理员可以撤销目标用户全部有效会话。

用户列表支持角色、状态、风险和排序筛选，并可批量执行恢复、停用、撤销会话和降为普通用户。批量操作最多处理 200 个用户，且不能影响当前管理员自己。

后台中的“用户影视记录”是用户与影视条目的数据库记录总数，不等于单一的点击次数。行为构成拆分为“想看、在看、看过、收藏、评分、短评”六项；同一条影视记录可以同时处于某个观影状态，并包含收藏、评分和短评，因此六项数据可能重叠，不能直接相加当作独立用户数。

“强制退出全部设备”会把目标用户当前所有未撤销的登录会话写入撤销时间，使其在浏览器、手机等设备上的登录状态失效。此操作不会删除账号、观影状态、收藏、评分、短评或历史记录。

用户列表和详情抽屉提供快捷角色操作：

- “提升管理员”会将普通用户提升为管理员并加入系统内置的“超级管理员”组，获得完整后台权限；执行前必须确认目标账号。
- “降级为用户”会将管理员改为普通用户，同时清除其管理员分组与用户级权限覆盖。
- 系统禁止管理员通过快捷操作降级自己。

### 8.5 分组和用户级权限

“分组与权限”可以新建自定义管理员分组，并配置 15 个权限点。用户详情中可以分配多个分组，并对单项权限设置继承、单独允许或明确拒绝。为防止当前操作人误锁死自己，管理员不能在用户详情中直接修改自己的分组和覆盖规则。

## 9. 短评审核

管理员可以把短评从 `published` 设为 `hidden`，或从 `hidden` 恢复为 `published`。

短评隐藏后：

- 不再显示在影视详情页公开短评中；
- 不再显示在该用户对外公开的记录中；
- 用户个人中心仍保留原记录；
- 评分、收藏和观影状态不受影响；
- 管理员可以恢复公开。

当前审核是可逆隐藏，不是物理删除；后台不能修改用户正文。建议对广告、恶意内容、违法内容、严重灌水或违反站点规则的短评执行隐藏。

审核操作会进入审计日志。当前界面没有独立审核备注框，复杂事件应使用外部工单或管理记录补充原因。

该审核功能已接入真实 MySQL 数据链路，不是仅修改前端显示。管理员将短评设为隐藏时，后端更新 `user_media_entries.review_status`；影视详情公开短评和公开用户主页都只查询 `published` 状态。项目验收已验证“发布可见 → 隐藏后消失 → 恢复后重新出现”的完整过程，并在验证后清理临时用户、短评和审计记录。

## 10. API Provider、数据源日志和 SQL 缓存

### 10.1 API Provider 管理

API 管理展示 TMDB、imdbapi.dev、Just One API 和后续自定义 Provider 的启用状态、凭据是否配置、七日调用量、成功率、延迟及最近测试结果。

管理员可以维护名称、基础地址、健康检查路径、超时、每日额度说明、文档地址和备注，并从服务器发起连通性测试。系统 Provider 不能删除，但可以停用；自定义 Provider 可以删除。

API 密钥不会写入数据库或通过后台返回。后台只显示“已配置/缺少凭据”，密钥仍必须通过服务器环境变量提供。内置 Provider 的启停、地址和超时会真实影响后续业务调用。

`provider_sync_logs` 记录 Provider、操作、缓存键、状态、耗时、错误和时间。后台显示最近 100 条同步日志。

第三方响应缓存全部写入 `api_cache`，不使用仅存在于 Node.js 进程内的业务缓存。缓存刷新不会修改用户数据。

缓存管理工作区支持按缓存键、Provider 和有效/过期状态分页查询，显示每条记录大小和时间；拥有 `cache.manage` 权限时可以删除单条、清空某个 Provider、清理全部过期缓存，或手动写入有效 JSON 缓存。清理后真实业务请求会重新访问上游接口，应注意调用额度。

手动检查：

```powershell
npm run cache:inspect
npm run cache:export
npm run cache:export -- cache-review.json
```

缓存导出可能包含大量第三方响应，不应提交到 Git 或公开发送。

| 现象 | 检查方向 |
| --- | --- |
| 大量 IMDb 评分暂无 | IMDb API、IMDb ID 映射和同步日志 |
| 首页或探索返回 `502` | Provider 超时、令牌额度、网络和 API 日志 |
| 新闻为空或图片失败 | 新闻 Provider、缓存和图片源 |
| 数据长期不更新 | 缓存过期时间、同步错误和系统时间 |
| 只有写请求失败 | `WEB_ORIGIN` 是否与真实 Origin 完全一致 |

## 11. 日志中心与审计

用户角色、账号状态和短评审核等操作会写入 `admin_audit_logs`，包含：

- 管理员 ID；
- 操作类型；
- 目标类型和目标 ID；
- 操作详情 JSON；
- 请求 IP；
- 操作时间。

后台显示最近 100 条。不要直接修改或清空审计表；如需归档，应先导出并验证备份，再按正式运维流程处理。

日志中心还可以查看 Provider 同步日志和登录安全记录，并按类型、关键词、Provider 和成功/错误状态筛选。登录记录只显示账号标识哈希的短指纹，不暴露用户输入的账号明文。

## 12. 站点设置

站点设置保存在 MySQL `site_settings`，当前包括：

- 开放注册：关闭后注册接口返回 403；
- 维护模式：开启后普通访客和用户访问业务 API 返回 503，管理员仍可进入后台排障；
- 站点公告：保存启用状态和公告文本，公开设置接口可供主站读取；
- 短评自动公开：关闭后新提交或修改的短评默认进入隐藏状态，等待管理员审核。

所有设置修改都会写入管理员审计日志。开启维护模式前应确认至少有一个有效超级管理员会话。

## 13. 认证安全

- 会话 Cookie 为 HttpOnly、`SameSite=Lax`；
- `NODE_ENV=production` 时自动加 `Secure`；
- MySQL 只保存会话 Token 的 SHA-256 摘要；
- 注销和修改密码会撤销会话；
- 停用用户无法继续鉴权；
- 密码采用随机盐 `scrypt`；
- SQL 使用参数化查询；
- 写请求执行 Origin 校验；
- 同一账号标识和 IP 在 15 分钟内失败 8 次后触发登录限流。

生产环境必须使用 HTTPS、准确配置 `WEB_ORIGIN`，并确保反向代理不会错误重写 Origin 和 Cookie。

## 14. 数据备份与隐私

数据库包含邮箱、密码哈希、IP、浏览历史、搜索历史和短评。管理员应：

- 定期备份 MySQL 并验证可恢复；
- 加密备份并限制访问；
- 不提交 `.env`、SQL 转储、缓存或日志导出；
- 排障截图遮盖邮箱、IP 和环境变量；
- 仅在必要范围内查询用户历史；
- 生产环境使用最小权限数据库账号。

当前后台没有数据库备份按钮，备份和恢复需要使用部署环境的 MySQL 运维工具。

## 15. 后台 API

| 方法 | 接口 | 用途 |
| --- | --- | --- |
| `GET` | `/api/v1/admin/overview` | 获取用户、互动、会话、错误和缓存概览 |
| `GET` | `/api/v1/admin/analytics` | 获取增长、用户、内容、搜索和 Provider 统计 |
| `GET` | `/api/v1/admin/users?query=` | 搜索用户 |
| `PATCH` | `/api/v1/admin/users/:id` | 修改角色或状态 |
| `POST` | `/api/v1/admin/users/bulk` | 批量用户操作 |
| `GET/POST/PUT/DELETE` | `/api/v1/admin/groups` | 管理后台分组与权限 |
| `PUT` | `/api/v1/admin/users/:id/access` | 设置用户管理员分组和覆盖规则 |
| `GET` | `/api/v1/admin/reviews?status=` | 查询短评，可筛选状态 |
| `PATCH` | `/api/v1/admin/reviews/:id` | 隐藏或恢复短评 |
| `GET/POST/PUT/DELETE` | `/api/v1/admin/providers` | 管理 API Provider |
| `POST` | `/api/v1/admin/providers/:key/test` | 测试 Provider 连通性 |
| `GET/PUT/DELETE` | `/api/v1/admin/cache` | 查询、写入和清理 SQL 缓存 |
| `GET` | `/api/v1/admin/logs` | 获取审计和 Provider 日志 |
| `GET/PUT` | `/api/v1/admin/settings` | 查看和修改站点设置 |

全部接口要求管理员会话；写接口还要求合法 Origin 和 JSON 请求体。前端隐藏按钮不能替代后端权限控制。

## 16. 常见故障

### 无法进入后台

确认账号已登录、`users.role` 为 `admin`、`users.status` 为 `active`，并在提升角色后重新登录。还需检查 API 是否运行、请求是否携带 Cookie。

### 后台请求错误

| 状态 | 含义 |
| --- | --- |
| `401` | 会话不存在或过期 |
| `403` | 非管理员、账号停用或 Origin 不合法 |
| `429` | 登录安全限流 |
| `500` | 数据库或内部服务错误 |
| `502` | 上游 Provider 或代理异常 |

“服务器返回的数据格式无效”通常表示代理端口错误、API 未启动，或服务返回了 HTML/空响应。

### MySQL 连接失败

检查 MySQL 服务、主机端口、数据库名称、账号权限和环境变量。配置 `MYSQL_URL` 时，它会优先于分项连接参数。

## 17. 管理员验收步骤

1. 普通账号访问 `/admin`，确认不能进入。
2. 使用脚本提升测试账号，重新登录后确认后台入口出现。
3. 核对用户、互动和有效会话概览。
4. 分别使用用户名、邮箱和显示名称搜索账号。
5. 停用普通账号，确认鉴权失败，再恢复账号。
6. 尝试停用当前管理员自己，确认后端拒绝。
7. 尝试移除自己的管理员角色，确认后端拒绝。
8. 普通用户发布公开短评，确认详情页可见。
9. 管理员隐藏短评，确认公开位置不再展示。
10. 恢复短评并确认重新公开。
11. 检查上述操作是否写入审计日志。
12. 检查 Provider 日志和 SQL 缓存统计。
13. 从非配置 Origin 发起写请求，确认返回 `403`。
14. 连续使用错误密码，确认第 9 次返回 `429`。
15. 执行项目验证：

```powershell
npm run typecheck
npm run build
node --test server/home-service.test.mjs server/user-service.test.mjs
```

16. 新建自定义权限组，配置权限后分配给测试管理员，确认菜单和 API 权限同步变化。
17. 对 Provider 执行健康测试，确认测试状态、延迟和同步日志更新。
18. 写入临时缓存并按键查询、删除，确认 MySQL 中无残留。
19. 关闭注册后验证注册接口返回 403，再恢复注册。
20. 关闭短评自动公开后提交测试短评，确认默认进入隐藏状态。
