# MovieScope 需求理解

> 当前基线：2026-07-12。本文件描述当前真实产品需求、已实现范围与约束，不保留已废弃方案。实现细节见《方案设计》和《项目与架构说明》；若文档与源码冲突，应同步修正文档和代码，而不是把本文件降级为历史记录。

## 1. 产品定义

MovieScope 是一个电影与剧集资料聚合、内容发现、口碑比较和个人评价平台。系统将 TMDB 的影视目录、IMDb 的国际评分与深度资料、豆瓣的中文评分与长评、以及 MovieScope 站内用户评价统一到同一套页面和数据模型中。

项目需要解决四类实际问题：

1. 影视资料分散：海报、剧照、预告片、演职员、角色、奖项、票房和评论通常分布在不同平台。
2. 中文与国际信息割裂：TMDB 适合主体资料，IMDb 适合国际评分与深度信息，豆瓣适合中文口碑。
3. 外部 ID 不统一：TMDB ID、IMDb ID、豆瓣 ID 不能直接互换，需要可靠映射和验证。
4. 计费接口成本：Just One API 按调用计费，必须按数据变化速度缓存并避免无意义请求。

## 2. 目标用户与价值

- 游客：无需登录即可搜索、浏览、筛选和查看多来源影视资料。
- 注册用户：可评分、撰写站内评价、标记剧透、维护个人资料并查看历史。
- 影视爱好者：比较 TMDB、IMDb、豆瓣口碑，查看完整演职员、剧照、奖项、票房和花絮。
- 内容研究者：使用结构化资料分析作品、人物、榜单、评价和票房。
- 管理员：管理用户、站内评价、权限、Provider、缓存、站点设置和审计日志。

项目可用于个人影视资料库、课程或毕业项目、文娱数据聚合原型、影视研究工具、内容运营后台和推荐产品的基础设施。

## 3. 产品原则

1. TMDB 是免费主体源，除明确指定外优先使用 TMDB。
2. IMDb 数据统一来自 Just One API，不使用 `imdbapi.dev`。
3. 豆瓣只展示评分、长篇影评和近期热门，不接入豆瓣短评。
4. 外部评分必须标明来源，不把不同平台评分直接平均。
5. 站内用户评价与外部评价分开存储、分开展示。
6. Just One 数据必须持久化缓存；不同数据采用与变化速度匹配的 TTL。
7. 外部接口失败时应显示可理解状态，并尽可能使用最近成功缓存。
8. 用户界面风格保持统一，不因新增数据改变现有主题和核心布局。
9. 浏览器不得持有第三方 Token，所有外部请求由后端代理和聚合。

## 4. 功能范围

### 4.1 首页

- 横向 Hero 推荐，展示中文标题、原文标题、IMDb 评分和背景图。
- 正在上映、热门电影、热门剧集、趋势与高分内容。
- IMDb Top 250 内容补充。
- IMDb 分类新闻。
- 首页默认热门来源为 TMDB，不自动调用豆瓣近期热门。

### 4.2 搜索与探索

- 搜索电影、剧集、影人和关键词。
- 按媒体类型、类型、年份、语言、地区、最低评分、观看平台和排序方式筛选。
- 支持热门、评分、发行时间和电影票房排序。
- 搜索结果展示海报、标题、年份、类型、评分和收藏状态。

### 4.3 榜单

- TMDB 正在上映、即将上映、热门电影、高分电影、热门剧集、高分剧集和今日播出。
- TMDB 日趋势与周趋势。
- IMDb 电影 Top 250。
- IMDb 剧集 Top 250。
- 热门电影/剧集可在 TMDB 与豆瓣来源间切换，默认 TMDB；切换到豆瓣时才调用付费接口。

### 4.4 影视详情

- Hero 使用横向背景图，默认保留标题、按钮和文本；播放预告片时隐藏叠加内容并在 Hero 区域播放。
- 中文标题下显示原文标题；有短经典语句时以斜体引用展示。
- 展示类型、年份、时长、分级、上映状态、国家、语言、公司、预算、票房和外部 ID。
- 展示导演、编剧等核心主创。
- 演员卡片明确显示“饰……”角色；查看更多进入完整演职员页面。
- 剧照区域展示接口返回的全部图片；点击进入完整剧照页面。
- 推荐与相似影视使用 TMDB，展示评分并保证合理数量。
- 媒体口碑展示 TMDB、IMDb、豆瓣评分及平台跳转。
- IMDb 票房摘要展示制作预算、首周末、北美累计和全球累计。
- 奖项展示总获奖、总提名、重要奖项与排名。
- “你知道吗”完整展示花絮、经典台词、穿帮、特殊演职员表和原声。

### 4.5 评价体系

评价分为：

1. 本站评价：MovieScope 用户发布，支持评分、正文、剧透标记、热度/时间和剧透筛选。
2. 影迷评价：平台可选 TMDB、IMDb、豆瓣，默认 TMDB；支持热度/时间和剧透筛选，默认五条并可加载到全部已获取数据。
3. 影评人评论：来自 Just One IMDb，默认五条，可逐步显示全部接口数据；上游无法继续分页时提供 IMDb 外链。

TMDB 评论必须拉取 `/movie|tv/{id}/reviews` 报告的全部分页，并保留评论 ID、作者、用户名、头像、评分、语言、完整正文、创建时间、更新时间和原评论 URL。TMDB 未提供点赞、回复和可靠剧透字段时保持空值，不伪造数据。

IMDb 用户评价必须尽可能完整展示用户、评分、正文、时间、语言、剧透和有用度。豆瓣只展示长评及其标题、作者、头像、评分、正文、时间、回复、转发、收藏等真实非零字段。

### 4.6 影人

- 热门影人列表和部门筛选。
- 影人详情展示头像、简介、出生/逝世信息、出生地、知名领域、外部 ID、照片和作品。
- 作品按热度组织并展示角色或职位、年份和评分。

### 4.7 用户系统

- 注册、登录、退出和当前会话查询。
- HttpOnly Cookie 会话。
- 修改个人资料、头像、横幅、简介和主页可见性。
- 修改密码后撤销现有会话。
- 站内 0.5–10 分评分、评价正文和剧透标记。
- 个人影视条目、浏览历史和搜索历史。
- 公开/私密用户主页。
- 当前详情页不展示“想看/在看/看过/收藏”按钮；数据库兼容字段可供个人中心和未来功能使用。

### 4.8 管理后台

- 管理员访问权限和细粒度权限判断。
- 概览、用户增长、内容和影视分析。
- 用户查询、详情、批量操作、状态和角色调整。
- 管理员组、组权限、用户组归属和用户级允许/拒绝覆盖。
- 站内评价审核与批量处理。
- TMDB/Just One Provider 配置、启停、超时和连接测试。
- API 缓存查询、写入、删除与统计。
- 站点设置：注册开关、维护模式、公告等。
- 管理操作审计日志。

## 5. 数据来源需求

### TMDB

负责搜索、发现、趋势、主体详情、中文标题、简介、类型、国家、语言、分级、公司、图片、预告片、演职员角色、影人、推荐、相似作品、观看平台、TMDB 评论和默认榜单。

### Just One IMDb

负责 Redux 评分与概览、用户评价、影评人评论、奖项、“你知道吗”、票房摘要、电影/剧集 Top 250 和新闻。

### Just One 豆瓣

负责豆瓣条目详情与评分、长篇影评、长评详情和近期热门电影/电视剧。豆瓣短评接口必须保持停用。

### Wikidata

优先把 IMDb ID 映射为豆瓣 ID。映射结果必须再通过豆瓣条目详情中的 IMDb ID 精确验证；失败后才使用标题和年份搜索候选。

## 6. 缓存与费用需求

- 通用单片数据：上映 ≤14 天为 1 天；14–60 天为 7 天；60–90 天为 30 天；>90 天为 60 天。
- 奖项：上映 ≤180 天为 7 天；之后为 45 天。
- “你知道吗”：上映 ≤30 天为 7 天；之后为 45 天。
- IMDb 电影/剧集 Top 250：7 天。
- 豆瓣近期热门：3 天。
- IMDb 新闻：1 天。
- 豆瓣完整长评正文：10 年。
- IMDb→豆瓣映射成功：180 天；失败：7 天。
- TMDB 评论分页：6 小时。
- 上游失败允许读取 365 天内最近成功缓存。
- Just One 不设置全局最低 TTL，不对业务错误进行额外重试。

## 7. 非功能需求

- 响应式：桌面、平板和手机均可使用。
- 可用性：统一 loading、empty、error、retry 状态。
- 性能：首页并行请求；列表分页；图片按尺寸加载；外部请求持久化缓存。
- 安全：密码哈希、HttpOnly Cookie、登录限流、请求体大小限制、CORS、权限校验和审计。
- 可维护性：服务按领域拆分；前后端 TypeScript 契约；数据库迁移可重复执行。
- 可部署性：Docker Compose、Nginx、GitHub Actions、GHCR 和健康检查。
- 可验证性：类型检查、Node 测试、生产构建和真实上游回归。

## 8. 验收标准

1. `npm run check` 全部通过。
2. 首页、搜索、探索、详情、影人、榜单、用户和后台主要路由可用。
3. IMDb 电影/剧集 Top 250 可访问。
4. 影视详情能展示可用的 IMDb 票房、奖项和“你知道吗”。
5. TMDB/IMDb/豆瓣影迷评价可切换，默认 TMDB，并能查看全部已拉取评论。
6. 豆瓣长评按真实互动字段排序和展示，不出现无意义的全零统计。
7. TMDB 评论不会误调用豆瓣长评详情接口。
8. Just One 调用命中 SQL 缓存，TTL 符合本文件规则。
9. 外部 Token 不出现在浏览器、仓库和日志中。
10. 文档中的接口、脚本、目录和配置与当前仓库一致。

## 9. 明确不做或已取消

- 不使用 `imdbapi.dev`。
- 不接入或展示豆瓣短评。
- 不把 TMDB、IMDb、豆瓣和站内评分混成单一分数。
- 不伪造上游未返回的点赞、回复、剧透或用户信息。
- 不在详情页展示“想看/在看/看过/收藏”按钮。
- 不把生产密钥、真实服务器配置、日志、数据库运行目录或构建产物提交到 Git。
