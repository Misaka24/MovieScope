# MovieScope 项目与架构说明

> 实现基线：2026-07-12。本文件描述当前真实源码；其他文档中的历史章节若与此冲突，以本文件和源码为准。

## 1. 定位、价值与实际问题

MovieScope 是多来源影视资料与个人观影管理平台。它解决中文主体资料、IMDb 深度信息、豆瓣中文口碑和站内用户数据分散的问题，并通过 IMDb/TMDB/豆瓣 ID 对齐、MySQL 缓存和按上映时间变化的 TTL 控制付费 API 成本。

项目可用于个人影视库、影视发现与研究、课程或毕业设计、文娱数据聚合服务、内容运营后台原型，以及奖项、评论、票房、演职员等资料分析。

## 2. 总体架构

```text
Vue SPA → MovieScope Node API → TMDB / Just One / Wikidata
                 │
                 └→ MySQL：用户、管理、缓存、映射、分析
```

浏览器只请求站内 `/api`。后端负责 Cookie 鉴权、参数验证、外部 API 聚合、字段归一化、缓存、降级和数据库访问。第三方 Token 不进入浏览器构建产物。

## 3. 技术实现

### 前端

Vue 3 Composition API + TypeScript + Vue Router；Vite 负责开发代理、热更新和构建；Tailwind CSS 与 scoped CSS 实现深色主题、响应式布局和动画。`src/services` 是统一 API 客户端，`src/types` 是稳定契约，页面不直接访问第三方 API。

页面包括首页、搜索、探索、榜单、新闻、电影/剧集详情、完整演职员、完整剧照、影人、登录注册、个人/公开主页、观看平台页和管理后台。

影视详情提供横向 Hero、预告片原位播放、IMDb/豆瓣/TMDB 数据、导演编剧、角色、剧照、剧情、奖项、票房、“你知道吗”、本站评价、TMDB 用户评论、IMDb 用户评价、豆瓣长评和影评人评论。TMDB 评论不建立独立栏目，而是与 IMDb、豆瓣统一收纳在“影迷评价”的评论平台胶囊中；默认展示 TMDB，可切换 IMDb 或豆瓣。TMDB 评论会拉取接口报告的全部分页，并保留评论 ID、作者名称、用户名、头像、评分、语言、完整正文、创建/更新时间和原评论 URL。

### 后端

后端使用 Node.js ESM 和原生 `node:http`。`server/server.mjs` 处理路由、Cookie、CORS、JSON 和错误边界；目录、首页、豆瓣、用户、管理员、Provider、数据库和缓存分别由独立模块负责。

主要数据链路：TMDB ID 是站内影视主 ID；先从 TMDB `external_ids` 获取 IMDb ID，再查询 Just One IMDb；豆瓣数据先做 IMDb→豆瓣映射；返回前端前统一转换为 MovieScope 字段。

### 数据库与安全

MySQL 8 使用 `mysql2/promise` 连接池。API 启动时自动执行 `db/migrations/*.mysql.sql`。主要表覆盖用户、会话、登录限流、影视互动、历史、管理员组与权限、审核、Provider、站点设置、API 缓存、同步日志、影视分析和外部 ID 映射。

密码采用随机盐 scrypt 哈希；会话 Token 只以哈希形式存库；浏览器使用 HttpOnly Cookie。登录接口有限流，管理员操作写审计日志。

## 4. 外部数据源与接入逻辑

### TMDB：免费主体源

负责搜索、发现、趋势、热门/高分/上映榜、中文标题和简介、类型、国家、语言、分级、公司、预算与 TMDB 票房、海报、横向背景、剧照、视频、演职员角色、导演编剧、影人、推荐和相似影视。推荐影视保持 TMDB 来源。

### Just One IMDb：付费深度源

- Redux Overview：IMDb 评分、票数、标题概览和 Metacritic；
- User Reviews Summary：用户、评分、完整正文、时间、剧透和有用度；
- Critics Review Summary：媒体、作者、分数、摘要与外链；
- Awards Summary：获奖、提名、重要奖项和排名；
- Did You Know：花絮、经典台词、穿帮、特殊演职员表、原声；
- Box Office Summary：预算、首周末、北美累计和全球累计票房；
- Chart Rankings：`TOP_250` 电影榜与 `TOP_250_TV` 剧集榜；
- News by Category：影视行业新闻。

票房正确路径为 `/api/imdb/title-box-office-summary/v1`。IMDb 数据已完全停用 `imdbapi.dev`，统一来自 Just One API。

### Just One 豆瓣：中文口碑源

负责豆瓣条目详情与评分、长篇影评、影评详情、近期热门电影和电视剧。豆瓣短评接口不再请求和展示；互动数据只有大于 0 时显示。

### IMDb→豆瓣映射

先查 Wikidata SPARQL；拿到候选豆瓣 ID 后请求豆瓣条目详情，要求其 IMDb ID 完全一致。失败时按中文标题、原文标题和年份搜索候选，再做相同校验。成功映射缓存 180 天，失败缓存 7 天。

## 5. 缓存与费用控制

所有外部响应写入 MySQL `api_cache`。缓存未过期直接返回；到期才请求上游；上游失败时可使用 365 天内最近一次成功响应。Just One 没有全局最低 TTL，也不对业务错误额外重试。

| 数据 | TTL |
| --- | --- |
| Redux、IMDb 用户/影评人评价、票房、豆瓣条目/长评列表，上映 ≤14 天 | 1 天 |
| 同类数据，上映 14–60 天 | 7 天 |
| 同类数据，上映 60–90 天 | 30 天 |
| 同类数据，上映 >90 天或日期未知 | 60 天 |
| 奖项，上映 ≤180 天 | 7 天 |
| 奖项，上映 >180 天或日期未知 | 45 天 |
| “你知道吗”，上映 ≤30 天 | 7 天 |
| “你知道吗”，上映 >30 天或日期未知 | 45 天 |
| IMDb 电影/剧集 Top 250 | 7 天 |
| 豆瓣近期热门电影/剧集 | 3 天 |
| IMDb 新闻 | 1 天 |
| 豆瓣长评完整正文 | 10 年 |
| stale 故障回退窗口 | 365 天 |

豆瓣热门只在用户主动切换来源时请求；首页默认 TMDB。豆瓣短评停用，直接减少按页付费调用。

## 7. 部署、运维与测试

生产环境由宿主机 Nginx 和 Docker Compose 分层运行：宿主机 Nginx 负责域名、TLS 和公网入口；Web 容器负责静态前端及 /api 反向代理；API 容器负责业务与上游聚合；MySQL 容器只在 Docker 内部网络提供持久化服务。公网不直接开放 API 或 MySQL 端口。

发布入口是 main 分支。GitHub Actions 先执行类型检查、构建和服务端测试，再并行构建 Web/API 镜像并推送 GHCR，随后通过 SSH 上传 Compose 与脚本。部署前对运行中的 MySQL 执行一致性备份，部署脚本拉取镜像、更新容器并检查 /api/v1/health。验证、镜像构建或健康检查失败时，该次发布不应视为成功。

前端和后端通过容器镜像更新；数据库通过增量 SQL migration 更新；真实环境变量、DNS、TLS 和宿主机 Nginx 不由 Git 提交自动覆盖。MySQL 数据由命名卷持久化，常规 up -d 和容器替换不会删除数据。详细首次部署、Secrets、回滚和故障处理见[GitHub Actions 部署手册](./GitHub-Actions部署手册.md)，整体流程见根目录 [README](../README.md#生产部署架构)。

本地使用 `npm run dev:all`。生产由 Docker Compose 启动 MySQL、API 和 Nginx Web；GitHub Actions 构建两个镜像推送到 GHCR，再 SSH 到服务器执行部署脚本。Nginx 托管 SPA 并代理 `/api`。

发布前执行 `npm run check`，包括 Vue/TypeScript 检查、Node 测试和 Vite 生产构建。自动测试覆盖 TTL 边界、IMDb 评分解析、首页去重、中文标题、密码哈希和管理员权限。真实外部接口仍需使用有效 Token 做回归。

## 8. MovieScope 站内 HTTP API

所有接口默认返回 `{ data, error }`。公开接口无需登录；“用户”接口要求有效会话；“管理员”接口还要求对应权限。

### 8.1 系统与认证

| 方法 | 路径 | 权限 | 作用 |
| --- | --- | --- | --- |
| GET | `/api/v1/health` | 公开 | API 与数据库健康检查 |
| GET | `/api/v1/settings/public` | 公开 | 注册状态、公告等公开设置 |
| POST | `/api/v1/auth/register` | 公开/受注册开关控制 | 注册并创建会话 |
| POST | `/api/v1/auth/login` | 公开 | 登录、限流校验并创建会话 |
| POST | `/api/v1/auth/logout` | 可匿名 | 撤销会话并清 Cookie |
| GET | `/api/v1/auth/me` | 可匿名 | 当前用户与管理员权限摘要 |

### 8.2 用户与个人数据

| 方法 | 路径 | 作用 |
| --- | --- | --- |
| GET/PUT | `/api/v1/me/profile` | 获取或更新自己的资料 |
| PUT | `/api/v1/me/password` | 修改密码并清除会话 |
| GET | `/api/v1/me/media` | 查询个人影视条目，支持筛选分页 |
| GET/PUT/DELETE | `/api/v1/me/media/:type/:id` | 查询、保存或删除单部影视的评分/评价等数据 |
| GET | `/api/v1/me/history` | 获取浏览与搜索历史 |
| POST | `/api/v1/me/history/browse` | 记录浏览 |
| POST | `/api/v1/me/history/search` | 记录搜索 |
| DELETE | `/api/v1/me/history/:kind` | 清空指定历史 |
| GET | `/api/v1/users/:username` | 获取公开用户主页；私密主页受限制 |

## 10. 子系统运行逻辑

### 详情聚合

前端请求 TMDB 主 ID；后端一次获取 TMDB 详情和 append 数据，解析 IMDb ID，调用 Redux；外部评价接口再并行加载 TMDB 全部分页评论、IMDb 评论/奖项/花絮/票房和豆瓣映射/长评。每个来源独立失败，页面按 status 渲染。

### 首页容错

首页来源使用 `safeSource` 独立执行；TMDB 区域、IMDb Top 250 或新闻失败时记录来源状态，其他分区继续返回。IMDb 评分按 IMDb ID 去重补全，避免同一条目重复计费。

### 用户互动

登录用户的评分和评价按 `(user_id, media_type, tmdb_id)` 幂等 upsert。站内公开评价从已发布条目读取；管理员可隐藏或恢复。浏览详情和搜索时异步记录历史。

### 管理权限

权限集合由管理员角色、组权限和用户覆盖计算；显式拒绝优先。后台路由在服务端校验权限并写审计日志，前端权限只负责展示控制。

### API Provider 管理

后台系统 Provider 只保留两个：`tmdb`（显示为 TMDB）和 `justone`（显示为 Just One API）。Just One 同时承载 IMDb 与豆瓣付费接口，不再拆成 `imdb` 或 `imdbapi-dev`。迁移会把旧 `imdb` Provider 的配置、缓存统计和同步日志转到 `justone`，再删除旧记录；凭据分别读取 `TMDB_ACCESS_TOKEN` 与 `JUSTONE_API_TOKEN`。

### 缓存与降级

Provider 调用先查未过期 SQL 缓存；同 key 并发请求共享 Promise；上游成功写入新值；失败读取 365 天 stale。后台可以查看和清理缓存，但不能绕过服务端权限。
